          タイプ1

          • 辞書指定のないクエリです。
          • /solr/sample01/suggest?wt=json&suggest=true&suggest.count=5&sugget.q=[query]
          • 設定ファイルの先頭にあるdefault辞書だけを使って結果を返します。
          • payloadカラムに収納したデータを結果として表示します。

          タイプ2

          • default辞書とroman1辞書を指定するクエリです。
          • /solr/sample01/suggest?wt=json&suggest=true&suggest.count=5&suggest.dictionary=default&suggest.dictionary=roman1&suggest.q=[query]
          • default辞書とroma1辞書の複数を使って結果を返します。
          • 両方の辞書でヒットする場合、同じレコードが2つ結果として同じレコードが複数返ってきます。

          タイプ3

          • タイプ2に山手線(=1)制約を追加したクエリです。
          • /solr/sample01/suggest?wt=json&suggest=true&suggest.count=5&suggest.dictionary=default&suggest.dictionary=roman1&suggest.cfq=1&suggest.q=[query]
          • タイプ2の結果をフィルタリングして山手線のものだけが返ってきます。

          タイプ4

          • default辞書とroman2辞書を指定するクエリです。
          • /solr/sample01/suggest?wt=json&suggest=true&suggest.count=5&suggest.dictionary=default&suggest.dictionary=roman2&suggest.q=[query]
          • roma2辞書では、「しながわ」のようなクエリでもヒットします。
          • roma2辞書では、「shin」でも、「sin」でもでもヒットします。
